# 撰寫中文學位論文與簡報

## Overview

彙整指導教授（舜哥）對學位論文與口試簡報的撰寫原則：章節編排、各章定位、段落與圖文規範。適用於撰寫或檢視中文論文（繁體）與對應的口試／進度簡報。

## When to Use

- 規劃或調整論文章節次序
- 撰寫「相關研究」「背景知識」「系統實作」「未來展望」等章節
- 檢查論文段落的條列式是否合理
- 製作或檢查論文口試／進度簡報的標題、排版、圖文比例
- 想確認某段內容該放在哪一章

**不適用於：**
- 非學位論文的一般技術文件（請改用 technical-writing-best-practices）
- 英文會議論文的格式規範

## 論文標準章節

論文可分為九大章，實際順序可依主題性質調整或整併：

1. 緒論 Introduction — 前言 Foreword、動機 Motivation、目的 Objective
2. 背景知識 Background
3. 相關研究 Related Work
4. 設計需求 Design Requirement
5. 設計方法 Approach
6. 實驗 Experiment
7. 評估 Evaluation
8. 結論 Conclusion
9. 未來展望 Future Work

## 章節撰寫原則（舜哥版）

**章節順序**
通常先寫相關研究、後寫背景知識；但有些論文性質需要先鋪陳背景知識再接相關研究，依題目判斷。

**相關研究 Related Work**
比較相關論文或工具。引用論文時以作者姓氏為主，不需放完整標題；若論文本身有命名（例如系統名稱）可直接用該名稱，對讀者較有記憶點。脈絡段落不要花太多篇幅鋪歷史，重點放在「相關領域應用」與「與本論文的比較」。

**背景知識 Background**
只寫純技術介紹。不要寫「這項技術在本研究中如何被應用」——應用面全部收進「系統實作」章節。

**段落與條列**
不要無謂的條列式，直接寫成段落即可。若要條列，每一項需有足夠的描述，不然像在灌頁數。不要過度分段落。

**未來展望 Future Work**
沒有要做的東西就放少一點，不要為了湊篇幅亂補。

## 論文與簡報共通規範

### 編排原則
- **由大至小**：先概述再細節
- **循序漸進**：讀者的理解曲線要平滑
- **承先啟後**：每一節要能連到前後節的脈絡

### 標題
- 標題需具備「該內文摘要」的性質，讀標題就能知道這頁或這段在講什麼
- 簡報製作時，**不應出現複數頁面使用相同標題**
- 大標題要能直接表達內容
- 使用宣示性的標題定義不同討論點（通常是動名詞）

### 探討面向要明確
撰寫前先決定這一段是「以人為主」或「以流程為主」：

- **以人的角度探討**：需讓讀者理解目前在談哪個角色、該角色與論文方法的關係，以及多角色之間的互動
- **以流程探討**：需引導讀者認出目前流程處於整體流程的哪個階段、前後銜接什麼

### 內文
- 主詞、受詞與語法需一致且明確
- 圖中文字與正文字體、大小需一致
- 多個物件可用堆疊的方式呈現

## Quick Reference

| 情境 | 原則 |
|------|------|
| 要不要寫條列 | 段落優先；條列時每項要有完整描述 |
| Related Work 引用 | 用作者姓氏或系統名，不放完整標題 |
| Related Work 寫歷史 | 從簡；重點放在相關應用與比較 |
| Background 章節內容 | 只放技術介紹，不寫應用 |
| 系統應用描寫位置 | 放在「系統實作」章節 |
| 簡報標題重複 | 禁止；每頁標題需唯一且貼合內文 |
| 大標題 | 直接表達內容、使用宣示性動名詞 |
| Future Work 長度 | 沒要做就寫少一點 |
| 章節順序 | Related Work 通常先於 Background；視題目而定 |
| 圖文字體 | 圖中字體與正文需一致 |

## Common Mistakes

- **背景知識章寫成系統實作**：把「我們如何用這個技術」寫進 Background。修正：應用描述全部搬到系統實作章。
- **相關研究章塞歷史脈絡**：花大半篇幅回顧技術歷史。修正：歷史從簡，重點放在相關應用與比較。
- **簡報多頁共用同一標題**：例如連續三頁都叫「系統架構」。修正：改成「系統架構：資料層」「系統架構：查詢流程」等具摘要性的標題。
- **為了條列而條列**：用 bullet 拆開本該是段落的內容，顯得在灌頁數。修正：改寫成段落；真要條列時每項要有完整描述。
- **圖中字體與內文不一致**：匯出圖表時字型或字級與內文不符。修正：統一字體與大小。
- **探討面向混淆**：同一段落同時談「角色互動」又談「流程階段」，讀者抓不到敘事主軸。修正：一段只聚焦一種探討面向。
